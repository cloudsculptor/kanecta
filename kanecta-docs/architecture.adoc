== Entry points

[cols="1,3"]
|===
| Entry point | Path

| Studio
| `kanecta-app-studio` → `kanecta-api-client` → `kanecta-api` → `kanecta-lib` + `kanecta-ai/kanecta-claude-lib`

| Hosted functions (inside Studio or kanecta-runner)
| user code → `kanecta-sdk` → `kanecta-api-client` → `kanecta-api` → …

| Runner (pipeline / standalone)
| `kanecta-runner` → hosted functions → `kanecta-sdk` → `kanecta-api-client` → `kanecta-api` → …

| MCP server
| `kanecta-mcp` → `kanecta-lib`

| CLI
| `kanecta-cli` → `kanecta-lib`
|===

== Storage adapters

`kanecta-lib` selects a storage adapter at runtime:

[cols="1,3"]
|===
| Adapter | When to use

| `kanecta-filesystem`
| Default. Local JSON files under `~/.kanecta/`. Human-readable, zero config.

| `kanecta-database` (+ `kanecta-postgres`)
| Remote/hosted mode. Structured SQL storage for items and metadata.

| `kanecta-s3`
| Remote/hosted mode. Object storage for binary assets/files. Independent of `kanecta-database`.
|===
